Regras de Negócio

Validações Críticas

🚸
Menor de Idade (<18): apenas transações do tipo Despesa são aceitas. Receitas são bloqueadas na camada de serviço.
🗑️
Deleção em Cascata: ao deletar uma Pessoa, todas as suas Transações são removidas automaticamente via FK CASCADE no EF Core.
🏷️
Categoria × Tipo: ao criar uma transação de tipo Despesa, apenas categorias com finalidade Despesa ou Ambas são aceitas. O mesmo vale para Receita.
✅
Valor Positivo: o campo Valor da transação deve ser um número maior que zero, validado via FluentValidation.
📏
Tamanho dos campos: Nome da pessoa máx. 200 chars. Descrição de Categoria e Transação máx. 400 chars.

Padrões de Desenvolvimento

🏛️
Clean Architecture: separação em 4 projetos: API, Domain, Application, Infrastructure. Dependências apontam sempre para dentro (Domain).
🔄
Repository Pattern: acesso ao banco abstraído em interfaces, facilitando testes e troca de provedor.
📦
DTOs: a API nunca expõe entidades do domínio diretamente — usa objetos de transferência mapeados com AutoMapper ou manualmente.
🔗
CORS configurado: a API permite requisições do front-end (localhost:5173) em desenvolvimento.
📖
Swagger habilitado: documentação interativa dos endpoints disponível em /swagger no ambiente de dev.
